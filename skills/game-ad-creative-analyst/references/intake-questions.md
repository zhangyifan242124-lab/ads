# 首次信息收集

只问完成分析所必需的信息。用户已经提供的信息不要重复问。

## 首次分析某个游戏

先建立轻量游戏档案，只收集稳定信息：

1. 游戏名或代号
2. 游戏类型和核心玩法
3. 主要卖点
4. 主要目标用户
5. 常投平台和常投地区
6. 绝对不能碰的表达或素材禁区
7. 有没有已知有效或已知失败的素材套路

不要把“本次优化目标”写进游戏档案。优化目标属于本次任务上下文。

## 什么时候询问游戏档案

命中以下情况，询问轻量游戏档案：

- 第一次分析某个游戏，且用户没有提供足够游戏描述。
- 素材代号、项目名、题材、美术或玩法与最近一次分析明显不同，例如从空战肉鸽变成武侠割草。
- 用户要求拆解视频内容、判断钩子和卖点、判断玩法承接或误导点。
- 用户要求检查审核风险，尤其涉及玩法真实性、赌博感、血腥、擦边、IP 风险。
- 用户要求输出下一版素材 brief。
- 用户要求判断数据表现，但该判断明显依赖游戏品类、核心玩法或历史基准。

只要用户发视频并要求“分析这条/这一批素材”，就先确认本次按哪个游戏档案分析。不要因为素材编号相似、画面看起来相似，直接判定是同一款游戏。

如果用户已经给出一句话游戏描述，可以先按临时档案分析，但必须在报告中标注“临时档案边界”。如果用户完全没有给游戏信息，先问轻量档案，不要直接给完整视频拆解。

## 项目切换确认

当看起来可能换游戏时，先用一句话确认，不要直接套用旧档案：

```text
这批素材看起来从上一款空战项目切到了新的武侠割草项目。是新游戏吗？如果是，我先按临时档案分析；你可以补一句核心玩法和主要目标。
```

当看起来像同一游戏的新皮肤、新主题或联动包装时，也要确认真实玩法和目标用户是否不变：

```text
这条看起来像 XKT 的新主题包装。我先确认一下：它和之前 XKT 是同一款游戏、同一核心玩法吗？如果是，我按 XKT 档案分析；如果只是美术换皮但人群或玩法有变化，需要补一句变化点。
```

如果用户没有补充但任务可以继续，按“临时游戏档案”分析，并在结论里标注：

```text
本次按临时判断：武侠割草/竖屏/技能清场爽感。若产品真实核心玩法不同，素材迭代结论需要调整。
```

## 什么时候不询问游戏档案

以下情况不要弹出完整问卷：

- 用户只是要看一份数据表，且问题可以用 CTR、CVR、CPI、CPM、CPC、ROAS、留存等指标先做诊断。
- 用户已经给了足够上下文，或者已有对应游戏档案。
- 用户明确说先不要建档案。
- 本次只需要做报表字段识别、数据卫生检查或简单账户动作建议。

这类情况下可以先分析，并加一句边界说明：

```text
我可以先基于数据判断前端买量和账户动作；如果要进一步给视频拆解、审核风险或下一版 brief，需要补游戏核心玩法和卖点。
```

## 推荐提问方式

```text
为了避免泛泛分析，我先建一个轻量游戏档案，只记录长期稳定信息：
1. 游戏类型和核心玩法是什么？
2. 主要卖点是什么？
3. 主要投放平台和地区通常是哪里？
4. 有哪些素材禁区，比如血腥、擦边、提现、赌博感、山寨 IP、误导玩法？
5. 有没有历史上跑得好或跑不好的素材套路？
```

如果用户正在等你分析具体素材，问题要更短：

```text
分析前我需要确认本次游戏档案：这条是已有项目，还是新游戏/新主题？
如果是已有项目，告诉我项目代号即可；如果是新项目，补 3 点就够：核心玩法、主要卖点、目标用户/地区。
```

如果用户已经给了“游戏一句话描述”，可以继续分析，但开头必须写：

```text
本次按临时游戏档案分析：【用户给的一句话】。未确认的信息包括目标用户、真实核心循环、素材禁区和历史基准，所以关于人群匹配、留存/ROAS原因和审核风险的判断是方向级。
```

## 每次分析素材时

收集本次任务上下文：

1. 本次平台、地区和投放端：Android / iOS / 双端 / 未知
2. 账户阶段：冷启动、放量、衰退、复测、素材探索
3. 本次优化师卡点或账户决策：CPI 太高、CTR 不差但装不动、ROAS 差、素材衰退、判断是否放量、判断是否停投
4. 素材文件或素材描述
5. 投放数据
6. 用户最想回答的问题

## 可选补充

只在有帮助时追问：

- 目标用户画像
- 当前素材表现基准，例如合格 CTR、CPI、ROAS
- 当前成功素材的共性
- 当前失败素材的共性
- 是否允许强误导包装
- 是否需要优先通过审核而不是追求点击
- 内部数据口径，例如 D0 ROAS、D1 ROAS、次留、付费率
